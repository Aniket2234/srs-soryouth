1. Introduction

1.1 Purpose

This Software Requirements Specification (SRS) document provides a comprehensive description of the Solar CRM (Customer Relationship Management) system for Soryouth Renewable Energy Pvt Ltd. The system is designed to manage the complete lifecycle of solar installation projects, from lead generation through installation, maintenance, and performance monitoring. This document is intended for:

  • Development team members
  • Project managers
  • Quality assurance testers
  • Stakeholders and business owners
  • System administrators
  • End users (sales executives, engineers, managers)

1.2 Scope

The Solar CRM System is a comprehensive web-based application that will:

  • Product Name: Soryouth Solar CRM
  • Product Mission: Streamline and automate all aspects of solar project management from lead acquisition to post-installation maintenance

Key Capabilities:

  • Manage complete customer lifecycle from lead to post-installation support
  • Track and monitor solar panel installations and system performance
  • Automate communication workflows across multiple channels (Email, WhatsApp, SMS)
  • Provide real-time analytics and business intelligence
  • Manage inventory, vendors, and procurement
  • Handle financial operations including invoicing, expenses, and subsidies
  • Enable field operations with GPS tracking and mobile access
  • Integrate with external systems (Google Reviews, RMS, Email systems)

Benefits:

  • Increased operational efficiency through automation
  • Better customer relationship management
  • Real-time visibility into business performance
  • Reduced manual errors and data duplication
  • Improved collaboration across teams
  • Enhanced decision-making through analytics

Out of Scope:

  • Manufacturing of solar panels
  • Direct hardware control of solar installations
  • Third-party accounting software (beyond basic invoicing)

2.4 Operating Environment

Client-Side Requirements:

  • Web Browsers: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
  • Screen Resolution: Minimum 1280x720 (responsive design supports mobile)
  • Internet Connection: Broadband (minimum 2 Mbps)
  • JavaScript: Enabled

Server-Side Requirements:

  • Operating System: Linux (Ubuntu 20.04 LTS or higher)
  • Runtime: Node.js 18.x or higher
  • Database: MongoDB 14.x or higher
  • Memory: Minimum 4GB RAM (8GB recommended)
  • Storage: Minimum 50GB SSD
  • Network: Static IP with HTTPS/SSL support

Mobile/Field Operations:

  • Mobile Browsers: iOS Safari, Android Chrome
  • GPS: Required for location tracking features
  • Camera: Required for document capture and site photos

2.5 Design and Implementation Constraints

  1. Technology Stack:

    • Frontend must use React with TypeScript
    • Backend must use Node.js with Express
    • Database must be MongoDB
    • UI must use Shadcn UI component library

  2. Security Constraints:

    • All data transmission must use HTTPS/TLS 1.2+
    • Passwords must be hashed using bcrypt
    • Session management required for authentication
    • Role-based access control (RBAC) mandatory

  3. Performance Constraints:

    • Page load time < 3 seconds
    • API response time < 500ms for 95% of requests
    • Support minimum 100 concurrent users
    • Real-time updates with < 2 second delay

  4. Integration Constraints:

    • Must use official Google Reviews API
    • WhatsApp integration via Business API only
    • Email integration via IMAP/SMTP protocols
    • GPS data must comply with privacy regulations

  5. UI/UX Constraints:

    • Color scheme: Solid green and white (no gradients)
    • Must be responsive (mobile, tablet, desktop)
    • Must support dark mode
    • Must be accessible (WCAG 2.1 Level A minimum)

  6. Data Constraints:

    • All monetary values in Indian Rupees (₹)
    • Date format: DD-MM-YYYY (Indian standard)
    • All timestamps in IST (Indian Standard Time)
    • Phone numbers in Indian format (+91)

7.6 API Authentication Flow

1. User Login
   POST /api/auth/login
   { username, password }
   → Validate credentials
   → Create session
   → Return JWT token

2. Authenticated Request
   GET /api/leads
   Header: Authorization: Bearer <JWT>
   → Validate token
   → Check permissions
   → Return data

3. Session Management
   - Token expiry: 8 hours
   - Refresh token: 30 days
   - Single session per user (SSO)
   - Logout invalidates token
